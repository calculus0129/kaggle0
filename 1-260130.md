깨달은 점: Kaggle에서 따로 LLM 사용에 대한 내용을 지원하고 있지는 않다!


즉, LLM은 아이디어 브레인스토밍용으로 적합하지만, 그걸 filtering 하는 것은 사용자 스스로가 해내야한다는거지? 궁금한 게 이런 insight를 얻기 위해 굳이 ML이 도움이 되는 건 딱히 없지 않아? 예를 들어서 algorithm problem solving에도 활용할 수 있고, 수학경시대회 대비에도 활용할 수 있고. 똑같은 것 같은데

1️⃣ 네 말이 맞는 지점 (중요)

맞아. 이건 사실이야:

- 알고리즘 문제
- 수학 문제
- 글쓰기
- 기획

👉 LLM은 다 브레인스토밍 도구로 쓸 수 있음
👉 필터링은 항상 인간 몫

그래서:

“LLM 판단 능력은 범용적이다”

2️⃣ 그런데 ML/Kaggle이 특별한 이유 (여기서 갈림)
핵심 차이 하나

ML 문제에는 ‘거짓말 못 하는 현실’이 있다.

3️⃣ 알고리즘 / 수학 문제의 구조적 한계

알고리즘 문제

- 정답 있음
- 테스트 통과 = 끝
- LLM이 틀려도: “아, 버그네” 하고 넘어감

👉 판단이 단순함

수학 경시

* 논리적으로 맞으면 끝
* 사고 과정은 개인적
* 데이터 품질 개념 ❌

👉 현실 제약이 없음

4️⃣ ML/Kaggle은 뭐가 다르냐면
ML 문제는 항상 이렇다

* 정답이 없음
* 데이터가 더럽다
* metric이 절대적이지 않다

성능이 “운”처럼 출렁인다

👉 LLM이 제일 그럴듯한 거짓말을 하는 환경

그래서:

* LLM의 위험성이 드러남
* 판단 실패가 누적됨
* “아, 이건 믿으면 안 되네”를 체감함
